An aggregation pipeline first becomes an execution plan, then becomes storage cursor work

MongoDB execution layer

Parse pipeline Validate stages and expressions.
Optimize Move filters, coalesce stages, expose indexable predicates.
Plan Select index scan, collection scan, sort, group, join shape.
Execute SBE or classic stages pull batches from storage cursors.
Goal: minimize keys examined, documents fetched, blocking memory, and spills.

WiredTiger retrieval layer

Index B-tree Traverse sorted keys and return record identifiers.
Collection B-tree Fetch BSON records when the plan is not covered.
Cache Use resident pages, read compressed pages through the filesystem cache when cold.
Snapshot Resolve visible updates from chains, disk image, or history store.
Eviction Keep cache under target while scans and writes compete.
Spill files Blocking stages may write temporary data outside WiredTiger data files.
Storage cost: each missing index or wide fetch turns optimizer choices into page traffic.
