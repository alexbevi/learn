Durability is a handoff from memory to log to checkpointed data files

T0

WT transaction commits

Collection and index updates become committed in memory and visible according to transaction and timestamp rules.

T1

Log record exists

The write-ahead log records modifications between checkpoints so recovery can replay committed changes.

T2

Journal sync boundary

Client acknowledgement depends on write concern and journaling behavior, not only on cache mutation.

T3

Checkpoint reconciles

Dirty pages are written as a transactionally consistent view of each B-tree and history store.

Crash

Recovery starts here

WiredTiger opens from the last checkpoint, replays log records after it, and rolls back unstable timestamped data when required.

Journal: protects changes since the last checkpoint.
Checkpoint: creates the durable base image.
Recovery: combines both, then enforces timestamp stability.
