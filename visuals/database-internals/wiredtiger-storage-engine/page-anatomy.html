WiredTiger stores MongoDB collections and indexes as row-store B-trees

B-tree shape

root page
keys + refs
internal page
keys + refs
leaf page
keys + values

Pages split as sorted key ranges grow.

Leaf page

recordId
BSON document for collection table
index key
key pattern plus recordId for secondary index table
insert
new entries can sit in in-memory insert lists before reconciliation
update
existing entries accumulate update structures

Update chain

newest value transaction id + timestamp decide visibility
older value may still be needed by a snapshot reader
disk image reconciled page version from data file
history store older durable versions when needed
